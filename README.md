## 概要

BitsKeep は個人事業・小規模法人向けの電子部品管理システムです。部品の登録・在庫追跡・発注・案件紐付けをブラウザ上で一元管理します。

**アクセス URL:** https://bits-keep.rwc.0t0.jp/

---

## 機能一覧

| 機能 | 説明 |
|---|---|
| 部品管理 | スペック・画像・データシート・仕入先を登録。フリーワード・分類・スペック範囲で検索 |
| 在庫管理 | 入庫・出庫・在庫履歴。棚卸しモードで実数と差分を管理 |
| 発注管理 | 在庫警告から発注リストを作成し、商社別 CSV で出力 |
| 案件管理 | 案件と使用部品の紐付け。Notion との双方向同期 |
| 保管棚管理 | 棚のグループ管理・廃止・復元 |
| 商社管理 | 仕入先・単価・リードタイム・送料無料閾値 |
| マスタ管理 | 分類・パッケージ分類・詳細パッケージ・スペック種別の CRUD |
| 部品比較 | 最大5部品の仕様・価格を横並び比較（差分ハイライト） |
| Altium 連携 | SchLib/PcbLib 登録・部品詳細からシンボル/フットプリント紐付け |
| 設計ツール | 抵抗網計算・エンジニアリング計算補助 |
| ユーザー管理 | 招待メール・ロール設定（管理者/編集者/閲覧者） |
| 操作ログ | 全 CRUD 操作の変更差分を記録・フィルタ閲覧 |
| CSV インポート | テンプレート形式で部品データを一括登録 |
| DB バックアップ | pg_dump ダウンロード・SQL ファイルによる書き戻し |

## はじめる前に

### ログインとアカウント

管理者から招待メールが届いたら、メール内のリンクからパスワードを設定してログインします。

### 権限（ロール）

ログイン後、ヘッダ右上に自分の権限バッジが表示されます。

| 表示 | ロール | できること |
|---|---|---|
| 管理（赤枠） | admin | すべての操作 |
| 編集 | editor | 部品・在庫・案件の登録・編集 |
| 閲覧 | viewer | 一覧・詳細の参照のみ |

権限を変更したい場合は管理者に連絡してください。

### 最初にやること

BitsKeep を使い始める前に、以下のマスタを登録してください。部品登録時にこれらが必要になります。

1. **分類を登録する** — 「マスタ管理 → 分類」で「抵抗」「コンデンサ」「FPGA」などを追加
2. **パッケージ分類と詳細パッケージを登録する** — 「マスタ管理 → パッケージ分類」で「SMD」「THT」を追加し、その配下に「0402」「SOT-23」などを追加
3. **スペック種別を登録する** — 「マスタ管理 → スペック種別」で「静電容量」「耐圧」などを追加。必要に応じて英語名・記号・alias・基準単位も設定
4. **商社を登録する** — 「商社管理」で仕入先を追加
5. **保管棚を登録する** — 「保管棚管理」で棚を追加

## ダッシュボード

ログイン後の起点となる画面です。

### 今日の確認事項

- **在庫警告件数** — 発注点を下回った部品の件数。クリックで在庫警告画面へ
- **最近アクセスした部品** — 直近で開いた部品へのショートカット

### グローバル検索

**Ctrl+K**（Mac は Cmd+K）でどのページからでも検索ランチャーを起動できます。部品名・型番でインクリメンタル検索し、Enter で詳細ページへ移動します。

### 業務別メニュー

ダッシュボード下部に「在庫入力」「在庫警告」「案件管理」などのショートカットが並んでいます。よく使う操作は並び替えて保存できます（ヘッダ右上「ホーム設定」）。

## 部品管理

### 部品を登録する

「部品一覧」→ 右上「+ 新規登録」から登録画面を開きます。

#### 入力項目

| 項目 | 必須 | 説明 |
|---|---|---|
| 型番 | ✓ | メーカー型番。重複チェックあり |
| 名称 | | 通称・社内管理名 |
| メーカー | | 製造元名 |
| 入手可否 | | 量産中 / EOL / 在庫限り / 非推奨 |
| 分類 | | 複数選択可。事前登録が必要（[マスタ管理](#マスタ管理)） |
| パッケージ分類 | | 大分類を選ぶと詳細パッケージが絞り込まれる |
| 詳細パッケージ | | 0402・SOT-23 など |
| 代表保管棚 | | この部品を通常置く棚（[保管棚管理](#保管棚管理)で事前登録が必要） |
| スペック | | 「+ スペック追加」でスペック種別を選び、`typ / 範囲 / 最大 / 最小 / 3値` の5種から値のコンテキストを選択して入力。入力中に「基底: 1000Ω / 表示: 1kΩ」のようなリアルタイムプレビューが表示される。候補にない種別は行内の「+」から追加可能 |
| 仕入先 | | 商社・商社型番・単価・購入単位・価格ブレーク。複数追加可 |
| 部品画像 | | jpg/png/webp、5MB まで |
| データシート | | PDF ファイル。複数添付可。各PDFに表示名を付けられる |
| カスタムフィールド | | 任意のキーと値を自由に追加 |

#### 保存

フォーム上部と下部に「保存」ボタンがあります。長いフォームを下まで入力した後、画面を上に戻らなくても保存できます。

#### データシート解析補助

現状の部品登録では、データシートからの補助入力に次の 3 系統があります。

- `ChatGPTで自動入力` — Tampermonkey userscript 経由で `PDF選択 -> 一時アップロード -> ChatGPT Web 解析 -> レビュー` まで自動化
- `ChatGPTから貼り付け` — ChatGPT で PDF を読ませた結果 JSON を貼り付けてレビューする
- `Geminiで解析` — API 利用を許容する環境だけの任意導線

どちらの導線でも、解析後はいったんレビュー用モーダルが開きます。ここで次を確認・修正してからフォームへ適用します。

- 基本情報: `型番` `メーカー` `通称` `説明`
- 分類候補: `component_types[]` からデコードした複数候補を既存分類へ紐付けて選択
- パッケージ候補: `package_names[]` からデコードした複数候補の中から、既存の `パッケージ分類 / 詳細パッケージ` へ紐付けた 1 件を選択
- スペック候補: `name / name_ja / name_en / symbol` を使って既存スペック種別へ照合し、未一致項目はレビュー画面の「+」から追加、削除

データシート上の元表記は `抽出名` として確認用に表示されますが、保存される正本は `spec_type_id` です。表記ゆれはスペック種別の alias に寄せて管理します。

`ChatGPTで自動入力` は `Tampermonkey` userscript を前提に、`PDF選択 -> BitsKeepへ一時アップロード -> ChatGPT Web 解析 -> 結果レビュー -> 保存` を自動化します。自動化に失敗した場合は、同じ画面から `ChatGPTから貼り付け` へ即切り替えられます。応答テキストのコピー補助もあるため、DOM 変更やログイン切れが起きても手動 fallback へ退避できます。

#### Tampermonkey 導入手順

1. `https://bits-keep.rwc.0t0.jp/tampermonkey/bitskeep-chatgpt-helper.user.js` を Tampermonkey へインストールする
2. `https://bits-keep.rwc.0t0.jp/*` と `https://chatgpt.com/*` への実行を許可する
3. ChatGPT にログインした状態で BitsKeep の部品登録画面を再読込する
4. PDF を選択して `ChatGPTで自動入力` を押す
5. 自動化が失敗したら案内モーダルから `ChatGPTから貼り付け` へ切り替える

#### userscript 更新手順

1. 部品登録画面の `ChatGPT自動解析` モーダルから `userscript を更新` を押す
2. Tampermonkey の更新確認画面で `再インストール` または `更新` を承認する
3. `再読込して反映` を押して部品登録画面へ更新版 userscript を反映する
4. 再読込後に helper 成功トーストが出れば更新完了。旧版のままなら更新ダイアログが再表示される

`userscript を更新` 導線はキャッシュ回避クエリ付き URL を開きます。さらに userscript 自体にも `@downloadURL` / `@updateURL` を埋め込み、Tampermonkey 側の更新元を固定しています。

#### 一時PDFの扱い

- ChatGPT 自動入力で選択した PDF は一時保存され、署名付き URL で ChatGPT タブへ渡されます
- 一時PDFの有効期限は 2 時間です
- 部品保存が成功すると一時PDFは正式なデータシートへ確定され、使い終わった token は再利用できません
- キャンセル時は BitsKeep 側から一時PDF破棄 API を呼んで消します

### 既存部品を複製して登録する

同じメーカーの別型番など、似た部品を素早く登録できます。

1. 複製元の部品詳細ページを開く
2. 「複製」ボタンをクリック
3. 型番など差分だけ修正して「保存」

### 部品を探す

#### 基本検索

部品一覧の上部にある検索バーで型番・名称・メーカーをキーワード検索できます。

#### 詳細フィルタ

「詳細条件」を開くと以下の条件で絞り込めます。

- 分類（複数選択可）
- 入手可否
- メーカー
- パッケージ分類 / 詳細パッケージ
- スペック種別 + 値の範囲（例: 静電容量 100nF〜10μF）
- 在庫状態（在庫あり / 在庫切れ / 警告中）
- 購入日の範囲

フィルタを適用した状態は「フィルタチップ」として表示され、1クリックで個別に解除できます。

#### お気に入り

部品カードの星アイコンでお気に入り登録できます。「お気に入りのみ」フィルタで一覧を絞り込めます。

#### 並び順

更新日時・登録日時・型番・名称・在庫数で並び替えできます。

### 部品を比較する

1. 部品一覧でカード左上のチェックボックスを選択（最大5件）
2. 画面下部に表示される「比較」ボタンをクリック
3. 比較画面でスペック・価格・在庫を横並びで確認

差分がある行はハイライト表示されます。「差異のある行のみ表示」チェックで絞り込めます。

### 部品詳細

部品詳細ページでは以下の情報を確認・操作できます。

- 基本情報・スペック・仕入先・画像・データシート
- 在庫数（新品 / 中古 別）と保管棚
- **入庫 / 出庫** ボタン（[在庫管理](#在庫管理)）
- 在庫履歴（入出庫のログ）
- 関連案件（この部品を使っている案件一覧）
- 類似部品（同じ分類・パッケージの部品候補）
- Altium リンク（シンボル名・フットプリント名）

## 在庫管理

### 入庫する

#### 部品詳細から入庫

1. 部品詳細ページ → 右上「入庫」
2. 以下を入力して「確定」

| 項目 | 説明 |
|---|---|
| 数量 | 入庫する個数 |
| 区分 | 新品 / 中古 |
| 保管棚 | 置く棚（複数棚への分配も可） |
| ロット番号 | 任意 |
| 備考 | 任意 |

#### 在庫入力画面からまとめて入庫

複数部品をまとめて入庫する場合はこちらが便利です。

1. ダッシュボード「在庫入力」または全機能一覧 →「在庫入力」
2. 部品を検索して追加
3. 各部品の数量・棚を入力
4. 「一括登録」で確定

### 出庫する

1. 部品詳細ページ →「出庫」
2. 数量・用途・備考を入力して「確定」

在庫が足りない場合はエラーが表示されます。

### 在庫履歴を確認する

部品詳細ページ下部の「在庫履歴」セクションに入出庫のすべてのログが表示されます。日時・操作者・数量・棚・備考を確認できます。

### 在庫警告

発注点（`min_stock`）を下回った部品は自動的に在庫警告として検出されます。

- ダッシュボードの警告バッジで件数を確認できます
- 全機能一覧 →「在庫警告」で一覧を確認できます

発注点は部品登録・編集画面の「在庫下限」フィールドで設定します。

### 棚卸しをする

実際の在庫数とシステムの在庫数を照合・修正します。

1. 「保管棚」→ 右上「棚卸しモード」をオン
2. 画面上部のオレンジのバーが表示されます（棚卸し中の目印）
3. 各棚の「実数」欄に実際の在庫数を入力（現在の在庫数がデフォルト表示）
4. 差分（±）がリアルタイムで表示されます
5. 「確定」で保存 — 差分が在庫に反映されます

棚卸しモード中は「棚卸しモード終了」ボタンで確定せずに終了できます。

## 発注管理

### 在庫警告を確認する

全機能一覧 →「在庫警告」で発注点を下回った部品の一覧を表示します。

各行には以下の情報が表示されます。
- 部品名・型番
- 現在の在庫数（新品 / 中古）
- 発注点との差分
- 発注済みで納品待ちの件数

### 発注リストを作る

1. 在庫警告画面で発注したい部品にチェックを入れる
2. 「発注リストへ追加」ボタンをクリック
3. 発注画面（全機能一覧 →「発注画面」）に部品が追加される

### 発注画面で内容を決める

発注画面では各部品について以下を設定します。

| 項目 | 説明 |
|---|---|
| 購入単位 | バラ / テープ / トレー / リール / 箱 |
| 購入数量 | 発注する個数 |
| 購入商社 | 仕入先を選択。選択すると商社型番と単価が自動入力される |

### 商社別 CSV を出力する

「〇〇商社 をCSV出力」ボタンで、その商社向けの発注シートを CSV で書き出します。

CSV の列: 部品名 / 型番 / パッケージ / 購入単位 / 商社 / 商社型番 / 数量 / 単価 / 小計

## 案件管理

### 案件を作成する

1. 全機能一覧 →「案件管理」→「+ 案件を追加」
2. 案件名・事業（Notion 事業 DB と紐付く）を入力して保存

### 使用部品を追加する

案件詳細ページの「使用部品」セクションで部品を検索して追加します。使用数も合わせて記録できます。

### Notion 同期

「連携設定」で Notion API トークンと事業データベース ID を設定すると、Notion 上の案件データを BitsKeep に同期できます。

- 同期は「案件管理」画面の「同期」ボタンで手動実行します
- 同期結果（成功 / 失敗 / 件数）が画面上に表示されます
- Notion で作成した案件は `origin: Notion` として取り込まれます

Notion 連携の設定方法は [Notion 連携設定](#notion-連携設定) を参照してください。

## マスタ管理

部品登録に必要な分類・パッケージ・スペック種別を管理します。

### 分類

「抵抗」「コンデンサ」「FPGA」など部品の種類を表すタグです。部品には複数の分類を付けられます。

- 一覧の ↑↓ ボタンで表示順を変更できます
- 使わなくなった分類は「アーカイブ」で候補から外せます（部品への紐付けは残ります）

### パッケージ分類・詳細パッケージ

パッケージは2階層で管理します。

- **パッケージ分類**（例: SMD、THT）— 大分類
- **詳細パッケージ**（例: 0402、0603、SOT-23）— パッケージ分類に属する個別形状

部品登録時はパッケージ分類を選んでから詳細パッケージを選びます。

### スペック種別

「静電容量」「耐圧」「周波数」など、検索・比較したい仕様項目を定義します。スペック種別には **日本語名 / 英語名 / 記号 / alias / 基準単位** を設定できます。

- 日本語名は画面表示と標準検索名に使います
- 英語名・記号・alias は、英語データシートや略記号から同じ種別へ照合するために使います
- 記号は HTML ではなく `h_FE` `V_CBO` のように保存します。`_` は下付き、`~` は上付き表示用の記法です
- 基準単位は 1 つだけ設定します（例: 静電容量 → F、電流 → A）

部品登録時の表示単位は、基準単位から自動で人間向けに読みやすい接頭語へ寄せます。例:

- `0.000001 A` は `1uA`
- `1000 Ω` は `1kΩ`
- `0.000000005 s` は `5ns`

登録画面ではスペック行に自由な「名前」欄はありません。必ずスペック種別を選び、候補にない場合だけその場の追加モーダルで種別を登録します。値は `typ / 範囲 / 最大 / 最小 / 3値` の5種を扱えます。内部検索は常に基準単位へ正規化して行うため、`1uA` と `1e-6A` は同じ値として扱われます。

## 困ったときは

| 困りごと | 参照先 |
|---|---|
| 部品が見つからない | [「部品を探す」](#部品を探す)の詳細フィルタでメーカー・パッケージ・スペック範囲を試す |
| 在庫数が実物と合わない | [「在庫履歴を確認する」](#在庫履歴を確認する)でログを確認 → [「棚卸しをする」](#棚卸しをする)で修正 |
| 商社を選択できない | [「商社を登録する」](#商社を登録する管理者のみ)で先に商社を登録する |
| パッケージが候補に出ない | [「パッケージ分類・詳細パッケージ」](#パッケージ分類詳細パッケージ)に追加する |
| 分類が候補に出ない | [「分類」](#分類)に追加する |
| スペック種別が候補に出ない | [「スペック種別」](#スペック種別)に追加する。部品登録・詳細編集のスペック行からも管理者ならその場で追加できる |
| スペックの単位が出ない | [「スペック種別」](#スペック種別)で基準単位を登録する |
| 入力を間違えて保存してしまった | 部品詳細の「編集」で修正 → 管理者は[「操作ログ」](#操作ログ)で変更履歴を確認できる |
| Notion 同期が失敗する | [「Notion 連携設定」](#notion-連携設定)でトークンと DB ID を再確認 |
| 発注リストが消えた | 発注画面は閉じても保持されるが、ブラウザの LocalStorage をクリアすると消える |
| 画面の表示が崩れる | ブラウザをハードリロード（Ctrl+Shift+R）する |
| DB を以前の状態に戻したい | [「バックアップから書き戻す」](#バックアップから書き戻す管理者のみ)を参照 |
| 権限を上げてほしい | [「ロールを変更する」](#ロールを変更する管理者のみ)を管理者に依頼 |

## 環境構築

> サーバーの構築・保守を担当する方向けの情報です。

### 動作環境

| 項目 | バージョン |
|---|---|
| OS | Ubuntu 22.04 |
| Web サーバー | Apache2 + PHP-FPM |
| PHP | 8.3 |
| Laravel | 12 |
| Vue | 3 |
| Tailwind CSS | 3 |
| PostgreSQL | 16 |

### 初期構築手順

```bash
git clone <repo> /web/documents/BitsKeep
cd /web/documents/BitsKeep/bits-keep

composer install --no-dev
npm ci && npm run build

cp .env.example .env
php artisan key:generate
# .env の DB_* / APP_URL / FILESYSTEM_DISK を設定

# ⚠ 実行前に接続先・件数・バックアップ・復元手段の4点を確認すること
php artisan migrate --force

php artisan storage:link
```

### コード変更後の反映

```bash
cd bits-keep
npm run build   # 必ずビルドしてからブラウザをリロード
```

### やってはいけない操作

| 操作 | 理由 |
|---|---|
| `composer run dev` の実行 | `public/hot` が生成されブラウザが Vite サーバーに接続しようとして CSS/JS が全滅 |
| `php artisan serve` の実行 | Apache と競合する |
| `.env` を SQLite に変えて `migrate --force` | 本番 DB が空になる（2026-04-12 事故の再現） |

### DB 変更前の必須チェック（4点）

1. 接続先確認（`.env` の `DB_HOST` / `DB_DATABASE` を目視）
2. 件数確認（`SELECT COUNT(*) FROM components` などで実データを確認）
3. バックアップ確認（ダウンロード済みのダンプがあるか）
4. 復元手段確認（`psql` で書き戻せる状態か）

### バックアップ（CLI）

```bash
# DB 取得
PGPASSWORD=postgres pg_dump -h 127.0.0.1 -p 5432 -U postgres bitskeep \
  | gzip > /tmp/bitskeep_$(date +%Y%m%d).sql.gz

# DB 書き戻し
zcat /tmp/bitskeep_YYYYMMDD.sql.gz \
  | PGPASSWORD=postgres psql -h 127.0.0.1 -p 5432 -U postgres bitskeep

# ファイル（画像・データシート）取得
tar -czf /tmp/bitskeep_files_$(date +%Y%m%d).tar.gz \
  /web/documents/BitsKeep/bits-keep/storage/app/public/

# ファイル復元
tar -xzf /tmp/bitskeep_files_YYYYMMDD.tar.gz -C /
```
