# 2026-04-08 高頻度小修正系ツール専門家相談

## 相談テーマ

BitsKeep のツール候補について、`高損害の大問題` ではなく、`高頻度で、地味だが、間違いやすく、毎回ちょっとずつ時間を食う作業` に絞って洗い出す。

## 相談条件

- 相談先: `CLAUDE.md` に定義されている4名の専門家サポーター
- 方針: 起動不能、保護協調、大規模依存関係のような大物は脇へ置く
- 禁止: 単純な公式電卓
- 重視点: `制約付きで一気に早くなるもの` `視点/向き/記法の取り違えを減らすもの`

## 共通質問

1. 高頻度・小修正系の新規提案を4案ください。各案ごとに `名前` `どんなミス/手戻りを減らすか` `最低入力` `返すべき結果` を書いてください。
2. 4案の中で、特に `導入コストに対して時間短縮効果が大きい` 順に 1位〜4位を付けてください。
3. この軸では、今までの専門家相談で何が見落とされていたかを一言で指摘してください。
4. UIはどうあるべきか。`5秒で使えること` を重視して、見せ方を提案してください。

## 専門家別Q&A

### 1. 中堅の第一線で働いている電子回路設計者

**Q1**

**A1**
1. `抵抗組合せ探索`
   - どんなミス/手戻りを減らすか: 単純な直並列ではなく、E系列、在庫、許容差、最大段数まで含めて候補を出す。`紙では合うが部品がない` `1個多い/少ない` `誤差で外れる` を減らす。
   - 最低入力: 目標抵抗値または比率、使える抵抗候補、許容差、個数制約。
   - 返すべき結果: 候補回路、誤差、段数、在庫優先度。
2. `コネクタ視点補助リファレンス`
   - どんなミス/手戻りを減らすか: オス/メス、前面/背面、mating face/solder side の取り違えを潰す。`向きを逆に見て配線した` `ピン番号を鏡像で読んだ` `毎回ネット検索して迷う` を減らす。
   - 最低入力: コネクタ規格名と型番。
   - 返すべき結果: ピン表、両面図、視点ラベル、誤読しやすい注意点。
3. `部品向き・極性リファレンス`
   - どんなミス/手戻りを減らすか: ダイオード、電解コンデンサ、トランジスタ、レギュレータ、IC の pin1 と極性を一発で確認する。`極性逆` `pin1逆` `フットプリント視点違い` を減らす。
   - 最低入力: 部品名か型番。
   - 返すべき結果: 向き、pin配置、正面/裏面の見え方、典型ミス。
4. `0Ω/未実装/ジャンパ整理`
   - どんなミス/手戻りを減らすか: リビジョン違い、オプション実装、ストラップ抵抗の有無を毎回確認するのを減らす。`rev差分を見落とす` `DNPを実装した` `ジャンパ状態を読み違える` を減らす。
   - 最低入力: BOM、版数、実装条件。
   - 返すべき結果: 実装状態一覧、版差分、未実装ルール、注意点。

**Q2**

**A2**
1位 `コネクタ視点補助リファレンス`  
2位 `抵抗組合せ探索`  
3位 `部品向き・極性リファレンス`  
4位 `0Ω/未実装/ジャンパ整理`

**Q3**

**A3**
見落とされていたのは、`壊れる大問題` ではなく `毎回の視点変換と記法変換` です。設計の時間は、計算そのものより `向き` `面` `表記` `候補` を毎回読み替えるところで溶けています。

**Q4**

**A4**
- 入口は1つの検索欄だけにして、型番か症状を入れたら即結果を出す
- コネクタ系は `front/rear` と `male/female` の切替を常時見える場所に置く
- 抵抗系は `目標値` `使える部品` `許容差` だけ先に入れ、候補3つを先頭表示する
- 返答は長文より `正しい向き` `候補` `注意` の3段で出す

### 2. 高齢だがトランジスタ黎明期から回路設計しているエンジニア

**Q1**

**A1**
1. `制約付き抵抗ネットワーク探索`
   - どんなミス/手戻りを減らすか: 分圧、プルアップ/プルダウン、終端、基準電圧の `どれなら成立するか` を毎回手計算して悩む時間を減らす。
   - 最低入力: 目標比率、使用可能抵抗のE系列、許容総電力、許容誤差、直並列の上限段数。
   - 返すべき結果: 成立候補の組み合わせ、誤差順位、総抵抗値、消費電力、最小修正候補。
2. `コネクタ視点補助リファレンス`
   - どんなミス/手戻りを減らすか: オス/メス、前/後ろ、mating face/solder side の取り違えを減らす。
   - 最低入力: コネクタ規格名、ピン番号、信号名、視点指定。
   - 返すべき結果: 正しいピン配図、視点ラベル、ミラー注意、誤読しやすいポイント。
3. `極性・向き・シルク照合チェッカ`
   - どんなミス/手戻りを減らすか: ダイオード、電解コンデンサ、LED、IC、コネクタの向き取り違えを減らす。
   - 最低入力: 部品名、フットプリント候補、シルク印字、実装向きルール。
   - 返すべき結果: 向きの一致判定、危険な回転、実装時の見え方、要注意箇所。
4. `ケーブル結線・ストレート/クロス判定`
   - どんなミス/手戻りを減らすか: 通信系コネクタやハーネスで、毎回ネット検索しながら配線を確認する時間を減らす。
   - 最低入力: コネクタ種別、相手機器、ピン機能、視点、結線目的。
   - 返すべき結果: 結線表、ストレート/クロス判定、視点別図、ありがちな誤配線。

**Q2**

**A2**
1位 `コネクタ視点補助リファレンス`  
2位 `極性・向き・シルク照合チェッカ`  
3位 `制約付き抵抗ネットワーク探索`  
4位 `ケーブル結線・ストレート/クロス判定`

**Q3**

**A3**
見落とされていたのは、`毎回ちょっと迷うだけだが、積算するとかなり時間を食う参照ミス` です。大物の失敗ばかり見ていると、こういう地味な損失を取りこぼします。

**Q4**

**A4**
- 最初の画面はツール名ではなく、`抵抗を組みたい` `コネクタを見たい` `向きを確認したい` `結線を見たい` という症状ボタンにする
- 入力は1画面で完結、出力は `答え` `視点` `注意点` を1枚で出す
- コネクタ系は `front/rear` と `mating face/solder side` の切替を最上段に置く

### 3. 若手だが Zynq など FPGA を使った開発が主務のエンジニア

**Q1**

**A1**
- `通信系コネクタ視点リファレンス`
  - どんなミス/手戻りを減らすか: オス/メス、前面/背面、mating face/solder side の取り違え。RJ45, USB, D-sub, JST, M12 あたりで毎回地味に時間を食う。
  - 最低入力: コネクタ規格名、極性、ピン数、視点指定、信号名。
  - 返すべき結果: ピン配図、両面表示、視点ラベル、誤配線注意、ストレート/クロス差分。
- `制約付き抵抗ネットワーク探索`
  - どんなミス/手戻りを減らすか: 直並列の組み合わせ、E系列、手持ち在庫、許容電力、必要比率の詰め直し。
  - 最低入力: 目標抵抗値または比率、許容誤差、使用可能部品、電力、個数制約。
  - 返すべき結果: 候補構成、誤差、部品点数、在庫適合、採用しやすさ順。
- `設定抵抗・ジャンパ・プルアップ監査`
  - どんなミス/手戻りを減らすか: 基板ごとに違う pull-up / pull-down / strap / jumper の設定忘れ。
  - 最低入力: 対象IC、設定ピン、基板の実装状態、推奨値、許容値。
  - 返すべき結果: 現状設定のOK/NG、矛盾、推奨実装、未接続時の既定状態。
- `ピン1・向き・フットプリント視点チェッカ`
  - どんなミス/手戻りを減らすか: IC の向き、ピン1、裏表、フットプリントの見間違い。
  - 最低入力: パッケージ名、ピン1マーク、採用フットプリント、基板側の視点。
  - 返すべき結果: 表裏の見分け、ピン番号対応、危険な見間違い例、実装前チェック。

**Q2**

**A2**
1位 `通信系コネクタ視点リファレンス`  
2位 `制約付き抵抗ネットワーク探索`  
3位 `設定抵抗・ジャンパ・プルアップ監査`  
4位 `ピン1・向き・フットプリント視点チェッカ`

**Q3**

**A3**
見落とされていたのは、`大きな失敗` ではなく `毎回の小さい迷いと視点ミス` です。ここは電卓ではなく、参照と判断の摩擦を消す領域でした。

**Q4**

**A4**
- 最上段に検索窓を置き、対象を `コネクタ / 抵抗 / ジャンパ / 向き` で即分岐する
- 画面は `1画面1目的` にし、視点切替はボタン1つで `前面/背面` `オス/メス` を切り替える
- 結果は表より先に `図` と `赤い注意` を出す

### 4. 電子工学の大学教授

**Q1**

**A1**
- `コネクタ視点補助リファレンス`
  - どんなミス/手戻りを減らすか: `オス/メス`、`前面/背面`、`mating face / solder side` の取り違えを減らす。毎回ネット検索して図の見方を確認する時間を潰せる。
  - 最低入力: コネクタ規格名、型番、ピン数、用途。
  - 返すべき結果: 前面図/背面図、ピン番号、信号名、向きの注意、誤読しやすい点。
- `DIP/ジャンパ/ストラップ設定逆引き`
  - どんなミス/手戻りを減らすか: アドレス設定、終端設定、モード設定、起動モードの見間違いを減らす。基板を触るたびに表を探す手間が消える。
  - 最低入力: 基板名、スイッチ本数、設定対象、目標モード。
  - 返すべき結果: 設定表、現状の状態、変更すべきビット、禁止組み合わせ、正しい写真向き。
- `ピン1・極性・マーキング照合チェッカ`
  - どんなミス/手戻りを減らすか: ICのピン1、ダイオードの向き、電解コンデンサの極性、LEDの向き、フットプリントの向き違いを減らす。
  - 最低入力: 部品名、パッケージ、フットプリント、シルク表示。
  - 返すべき結果: ピン1位置、極性、表面実装時の向き、危険な誤読パターン。
- `制約付き抵抗合わせ込み`
  - どんなミス/手戻りを減らすか: 抵抗の直並列、分圧、プルアップ、終端、ゲイン設定を毎回手計算し直す時間を減らす。単純公式ではなく、E系列・在庫・許容差・電力制約込みで探す。
  - 最低入力: 目標値、許容誤差、使用可能なE系列、在庫候補、電力制約。
  - 返すべき結果: 候補組み合わせ、誤差、最悪値、推奨順、捨てるべき案。

**Q2**

**A2**
1位 `コネクタ視点補助リファレンス`  
2位 `DIP/ジャンパ/ストラップ設定逆引き`  
3位 `ピン1・極性・マーキング照合チェッカ`  
4位 `制約付き抵抗合わせ込み`

**Q3**

**A3**
見落とされていたのは、起動不能のような大物ではなく、`視点` `向き` `設定表` `E系列` のような、毎回じわじわ時間を奪う参照作業です。

**Q4**

**A4**
- 画面の上部に `コネクタ` `ジャンパ設定` `極性確認` `抵抗合わせ` の4ボタンを置き、押した瞬間に「型番入力1個 + 視点切替1個」だけ出す
- 結果は1画面で完結させ、`前面/背面` `オス/メス` `ミラー表示` はトグル1つで切り替える
- 検索結果には必ず `図 + ピン番号 + 注意点` を並べ、式ではなく見間違い防止を主役にする

## 横断的な読み取り

- 4名とも `コネクタ視点補助リファレンス` を最上位かそれに近い候補として挙げた
- `抵抗組合せ/制約付き抵抗ネットワーク探索` も 4名すべてが高評価した
- `DIP/ジャンパ/ストラップ設定逆引き` と `設定抵抗・ジャンパ・プルアップ監査` は、表現は違うが同じ痛みを指している
- `ピン1・極性・向き・シルク照合` も複数人から挙がった
- 見落とされていたのは `高損害の大問題` ではなく、`毎回の小さい視点変換・記法変換・参照ミス` だった
