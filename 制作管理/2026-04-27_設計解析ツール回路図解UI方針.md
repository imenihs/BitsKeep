# 設計解析ツール回路図解UI方針

作成日: 2026-04-27

## 背景

設計解析ツールでは `R1` `R2` `R3` `Rs` `Vref` のような回路記号を入力欄や計算結果で使っているが、現在の画面だけでは「その部品が回路上のどこにあるか」が分かりにくい。これでは利用者が式の意味を理解する前に止まり、単なる公式置き換えツールに見えてしまう。

特に、抵抗分圧、比較器ヒステリシス、電流検出、保護回路、周波数応答では、同じ `R1` でも役割が変わる。部品記号だけを見せるのではなく、回路上の位置、信号の流れ、計算式、入力欄を結び付ける必要がある。

## 結論

設計解析ツールは、回路要素を扱う場合に必ず `回路コンテキストパネル` を持つ。`R1` や `Rs` のような記号は、入力欄単体では出さず、役割名と簡易回路図に対応付けて表示する。

完成扱いにする条件は次の通り。

- 画面上部または入力欄横に、対象回路の簡易回路図またはブロック図がある
- 図中の `R1` `R2` `Rs` などの記号が、入力欄のラベルと一致している
- 入力欄は `R1 (Ω)` だけでなく、`R1 上側抵抗` `R2 下側抵抗` `Rs シャント抵抗` のように役割名を含む
- 計算式は、図の部品記号と同じ名前で表示する
- トポロジが複数あるツールでは、選択したトポロジに合わせて図と式が切り替わる
- 図で表せない条件は、仮定条件として明示する

## UI方針

### 回路コンテキストパネル

- SVGで描く。小さな回路図は画像化せず、拡大縮小、テーマ切替、入力欄とのハイライト連動を可能にする。
- 図は精密なCAD図ではなく、計算対象を理解するための最小構成にする。
- 信号名、電源名、GND、出力、測定点を必ず表示する。
- 入力欄にフォーカスしたとき、対応する図中要素を強調表示する。
- 図中要素をクリックしたとき、対応する入力欄へ移動する。
- モバイルでは図を入力欄の上に置き、横スクロール前提にしない。

### 入力ラベル

部品記号だけのラベルは禁止する。最低限、次の形式にする。

| 悪い例 | 良い例 |
|---|---|
| `R1 (Ω)` | `R1 上側抵抗 (Ω)` |
| `R2 (Ω)` | `R2 下側抵抗 (Ω)` |
| `R3 (Ω)` | `R3 帰還抵抗 (Ω)` |
| `Rs (Ω)` | `Rs シャント抵抗 (Ω)` |
| `C (F)` | `C フィルタ容量 (F)` |

### 式と判定

式は記号だけでなく、図のどの値から何を求めたかを短く添える。

例:

```text
Vout = Vin * R2 / (R1 + R2)
R1: Vin側の上側抵抗
R2: GND側の下側抵抗
```

判定パネルでは、数値結果だけでなく「どの部品を変えると改善するか」を図中の記号と同じ名前で示す。

## 対象ツール別の図解方針

| ツール | 必要な図解 |
|---|---|
| センサ分圧/NTC | Vin、R1、NTC/R2、Vout、ADC入力、温度スイープ対象 |
| 比較器ヒステリシス | Vin、Vref、R1入力抵抗、R2基準側、R3帰還、出力、上下しきい値 |
| 電流検出 | 負荷、Rs、検出アンプ/ADC、ケルビン接続、許容損失 |
| OVP/TVS/ヒューズ/ポリスイッチ | 入力、保護素子、負荷、故障電流経路、保護順序 |
| 周波数応答 | ローパス/ハイパス/2次系の選択トポロジ、入力、出力、R/C位置 |
| 電源余裕 | 電源レール、レギュレータ、負荷、dropout、突入/最大負荷の見える化 |
| 起動診断 | レール依存グラフ、PG/RESET、部分給電、逆流経路 |
| 熱設計 | 発熱源、パッケージ、TIM、ヒートシンク、周囲温度の熱抵抗チェーン |
| コネクタ/ケーブル | mating face/solder side、ピン1、反転方向、ストレート/クロス対応 |

## 実装方針

初期実装では、`design-tools` 内にツール別の図定義を持たせる。後で共通化できるよう、次の構造に寄せる。

```text
diagram = {
  topology,
  nodes,
  components,
  labels,
  fieldMap,
  equations,
  assumptions
}
```

`fieldMap` は、入力欄のキーと図中要素を結び付けるために使う。計算ロジックとは分離し、図のために計算式を重複実装しない。

実装順は次の通り。

1. `CircuitContextPanel` 相当の共通表示部品を追加する
2. 分圧/NTCと比較器を最優先で図解化する
3. 電流検出、OVP、TVS、ヒューズ、ポリスイッチを故障電流経路つきで図解化する
4. 周波数応答、電源余裕、熱設計、起動診断をブロック図またはチェーン図で図解化する
5. 入力欄フォーカスと図中要素ハイライトを連動させる
6. 解析結果の次アクションから図中要素へ戻れるようにする

## 完了判定

チェックリスト上の各設計解析ツールは、計算結果と判定パネルだけでは完了にしない。回路記号を含むツールは、図解、役割付き入力ラベル、式、仮定条件、ハイライト連動まで入って初めてUIとして完了扱いにする。
