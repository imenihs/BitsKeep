## [曹長(コード分隊)] 2026-04-10 — integration-settings.js レスポンス読み取りバグ修正

### 調査結果
「連携設定画面でトークンが未設定と表示される」問題を調査。

**根本原因:** `api.js` は fetch ラッパーで JSON を直接返す（axios形式ではない）。
- `r` = `{ success: true, data: { configured: true, ... }, message: '' }`
- `r.data` = `{ configured: true, ... }` ← 正しいデータ
- `r.data?.data` = `undefined` ← そのようなネストは存在しない

`integration-settings.js` の4箇所が `r.data?.data ?? notion.value` というパターンで、`?? r.data` フォールバックが欠落していた。結果、常に `undefined ?? notion.value` = 初期値 `{ configured: false }` のまま更新されなかった。

`project-list.js` は `r.data?.data ?? r.data ?? syncConfig.value` と正しく書かれていたため問題なし。

**修正:** `integration-settings.js` の4箇所を `r.data?.data ?? r.data ?? notion.value` に統一。

曹長の心の声: `api.js` が axios ではなく fetch 直書きのため `r.data` の意味が違う。同一プロジェクトで混在すると必ずこの罠に落ちる。fetch ラッパーなら `r` = JSONボディ、axios なら `r.data` = JSONボディ。今後も書く際に注意が必要。
