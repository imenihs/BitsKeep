## 1. 持っている機能

| 機能 | 概要 | 主な画面 |
|---|---|---|
| 部品管理 | 部品の登録・編集・画像/データシート添付・スペック管理・複製 | 部品一覧 / 部品詳細 / 部品登録 |
| 部品比較 | 最大5部品の仕様・価格を横並び比較（差分ハイライト） | 部品比較 |
| 在庫管理 | 入庫・出庫・棚割り当て・在庫履歴・在庫警告 | 部品詳細 / 在庫入力 |
| 棚卸し | 棚卸しモードで実数入力→差分調整 | 保管棚管理 |
| 発注管理 | 在庫警告から発注リスト作成・商社別 CSV 出力 | 在庫警告 / 発注画面 |
| 商社管理 | 仕入先・購入単価・リードタイム・送料無料閾値 | 商社管理 |
| 保管棚管理 | 棚のグループ管理・廃止・復元 | 保管棚管理 |
| マスタ管理 | 部品分類・パッケージ分類・スペック種別の CRUD | マスタ管理 |
| 案件管理 | 案件と使用部品の紐付け・Notion 同期 | 案件管理 |
| Altium 連携 | SchLib/PcbLib 登録・部品詳細からシンボル/フットプリント紐付け | Altium 連携 |
| 設計ツール | 抵抗網計算・エンジニアリング計算補助 | 設計ツール |
| ユーザー管理 | 招待・ロール設定（管理者/編集者/閲覧者） | ユーザー管理 |
| 操作ログ | 全 CRUD 操作の履歴閲覧・フィルタ | 操作ログ |
| CSV インポート | テンプレート形式で部品をまとめて一括登録 | CSV インポート |
| DB バックアップ | pg_dump ダウンロード・SQL アップロードによる書き戻し | DB バックアップ |

## 2. ログインとアカウント

### 初回ログイン

管理者から招待メールが届いたら、メール内のリンクからパスワードを設定してログインします。

### 権限について

ログイン後、画面右上に自分の権限が表示されます。権限を変更したい場合は管理者に連絡してください。

| 権限 | できること |
|---|---|
| 管理（赤枠） | すべての操作 |
| 編集 | 部品・在庫の登録・編集 |
| 閲覧 | 一覧・詳細の参照のみ |

## 3. 画面の見方

### ヘッダ

- 左の「BitsKeep」ロゴ → ダッシュボードへ戻る
- 「使い方」→ このガイド
- 「全機能一覧」→ すべての画面へのショートカット
- 右上のロールバッジ → 自分の権限表示

### ダッシュボード

- **今日の確認事項** — 在庫警告件数と最近アクセスした部品
- **業務別メニュー** — 在庫入力・発注・案件管理などのショートカット
- **Ctrl+K** — どのページからでも使えるグローバル検索

## 4. 部品を登録する

### 新規登録

1. 「部品一覧」→ 右上「+ 新規登録」
2. 以下の項目を入力します

| 項目 | 説明 |
|---|---|
| 型番 | メーカー型番（必須） |
| 名称 | 通称・管理名 |
| 分類 | 抵抗・コンデンサ・IC など（[8章](#8-マスタを整備する)で事前登録が必要） |
| パッケージ | 0402・SOT-23 など（[8章](#8-マスタを整備する)で事前登録が必要） |
| メーカー | 製造元名 |
| 入手可否 | 量産中 / EOL / 在庫限り / 非推奨 |
| 代表保管棚 | この部品を通常置く棚（[8章](#8-マスタを整備する)で事前登録が必要） |
| スペック | 容量・耐圧・周波数など（複数追加可） |
| 仕入先 | 商社名・商社型番・単価（複数追加可。[8章](#8-マスタを整備する)で事前登録が必要） |
| 画像 | 部品外観写真（jpg/png/webp、5MB まで） |
| データシート | PDF ファイル |

3. 「保存」で確定

### 既存部品を複製して登録

部品詳細ページの「複製」ボタンを使うと、型番と差分だけ修正して素早く登録できます。

### 部品を探す

部品一覧の検索・フィルタが使えます。

- キーワード検索（型番・名称・メーカー）
- 分類・パッケージ・スペック値の範囲絞り込み
- 在庫状態（在庫あり・切れ・警告中）での絞り込み
- お気に入り登録した部品だけ表示

### 部品を比較する

一覧でチェックボックスを選び（最大5件）、「比較」ボタンで仕様・価格を横並び表示します。差分がある行はハイライト表示されます。

## 5. 在庫を管理する

### 入庫する

**部品詳細から:**
1. 部品詳細ページ → 右上「入庫」
2. 棚・数量・新品/中古・ロット番号（任意）を入力して確定

**まとめて入庫:**
1. ダッシュボード「在庫入力」または全機能一覧 →「在庫入力」
2. 部品を選んで数量・棚を一括入力

### 出庫する

部品詳細ページ → 「出庫」ボタン → 数量・用途を入力して確定。

### 在庫履歴を確認する

部品詳細ページ下部の「在庫履歴」で、いつ誰が入出庫したかのログを確認できます。

### 棚卸しをする

1. 「保管棚」→ 右上「棚卸しモード」をオン
2. 画面上部のオレンジのバーに実際の在庫数を入力
3. 「確定」で保存（システム在庫と差分があれば調整されます）

## 6. 発注する

### 在庫警告を確認する

ダッシュボードの「在庫警告」バッジ、または全機能一覧の「在庫警告」で、発注点を下回った部品の一覧を確認します。

### 発注リストを作って CSV 出力する

1. 在庫警告画面で発注したい部品にチェックを入れる
2. 「発注リストへ追加」で発注画面に送る
3. 発注画面で商社・購入単位・数量を設定
4. 「〇〇商社 CSV出力」で商社ごとに発注シートを出力

---

## 7. 案件に部品を紐付ける

### 案件を作成する

全機能一覧 →「案件管理」→「+ 案件を追加」で案件名と事業を入力します。

### 使用部品を追加する

案件詳細の「使用部品」セクションから部品を検索して追加します。使用数も記録できます。

### Notion と連携する

「連携設定」で Notion API トークンと案件データベース ID を設定すると、Notion の案件データを同期できます。設定方法は「連携設定」画面内の案内を参照してください。

## 8. マスタを整備する

部品登録時に選ぶ「分類」「パッケージ」「スペック種別」「商社」「保管棚」は、あらかじめ登録が必要です。

| マスタ | 画面 | 何を登録するか |
|---|---|---|
| 部品分類 | マスタ管理 → 分類 | 抵抗・コンデンサ・FPGA など |
| パッケージ分類 | マスタ管理 → パッケージ分類 | SMD・THT など大分類 |
| 詳細パッケージ | マスタ管理 → 詳細パッケージ | 0402・SOT-23 など個別形状 |
| スペック種別 | マスタ管理 → スペック種別 | 容量・耐圧・周波数 など |
| 商社 | 商社管理 | 仕入先・購入先の会社情報 |
| 保管棚 | 保管棚管理 | 棚の名称・グループ・番号 |

### アーカイブと復元

使わなくなったマスタは「廃止（アーカイブ）」にすると選択候補から消えます。過去のデータには影響しません。「復元」でいつでも元に戻せます。完全に消したい場合だけ「完全削除」を使います。

## 9. ユーザーを追加する（管理者のみ）

1. 全機能一覧 →「ユーザー管理」
2. 「招待」ボタンでメールアドレスとロールを入力して送信
3. 相手が受信メールのリンクからパスワードを設定してログイン完了

ロールの変更は、ユーザー管理画面の一覧から「変更」ボタンで行えます。

## 10. 困ったときは

| 困りごと | 対処方法 |
|---|---|
| 部品が見つからない | [4章「部品を探す」](#4-部品を登録する)の詳細検索（メーカー・パッケージ・スペック範囲）を試す |
| 在庫数が実物と合わない | [5章「在庫履歴を確認する」](#5-在庫を管理する)でログを確認し、[棚卸し](#棚卸しをする)で修正 |
| 商社を選択できない | [8章「マスタを整備する」](#8-マスタを整備する)で商社を先に登録する |
| パッケージが候補に出ない | [8章](#8-マスタを整備する)の「マスタ管理 → パッケージ分類 → 詳細パッケージ」に追加する |
| 分類が候補に出ない | [8章](#8-マスタを整備する)の「マスタ管理 → 分類」に追加する |
| 入力を間違えて保存してしまった | 部品詳細の「編集」で修正する。管理者は[11章「操作ログ」](#11-管理者向け操作)で変更履歴を確認できる |
| Notion 同期が失敗する | [7章](#7-案件に部品を紐付ける)の「連携設定」で Notion トークンと DB ID を確認する |
| 画面の表示が崩れる | ブラウザをハードリロード（Ctrl+Shift+R）する |
| DB を以前の状態に戻したい | [11章「DB をバックアップ・復元する」](#11-管理者向け操作)から書き戻す |
| 権限を上げてほしい | 管理者に連絡する（[9章](#9-ユーザーを追加する管理者のみ)参照） |

## 11. 管理者向け操作

### 操作ログを確認する

全機能一覧 →「操作ログ」で、誰がいつ何を変更したかを確認できます。日付・操作者・操作種別でフィルタできます。

### CSV で部品をまとめて登録する

1. 全機能一覧 →「CSV インポート」
2. テンプレートをダウンロードして部品データを記入
3. ファイルをアップロードしてインポート実行

### DB をバックアップ・復元する

全機能一覧 →「DB バックアップ」で操作します。

**バックアップ取得:** 「📥 ダウンロード」ボタンで現時点の DB 全体を `.sql.gz` で保存します。定期的に手元に保管してください。

**書き戻し:** 保存済みの `.sql` または `.sql.gz` ファイルをアップロードすると、現在の DB に上書き反映されます。**元に戻せない操作**なので、書き戻す前に必ず最新のバックアップを取得してください。

### Altium ライブラリを管理する

全機能一覧 →「Altium 連携」で `.SchLib` / `.PcbLib` のパスを登録します。部品詳細ページからシンボル名・フットプリント名をライブラリと紐付けできます。

## 13. システム管理者向け構築・運用手順

> 以降はサーバーの構築・保守を担当する方向けの情報です。

### 動作環境

| 項目 | 内容 |
|---|---|
| OS | Ubuntu 22.04 |
| Web サーバー | Apache2 + PHP-FPM |
| PHP | 8.3 |
| フレームワーク | Laravel 12 / Vue 3 / Vite / Tailwind CSS 3 |
| DB | PostgreSQL 16 |
| URL | https://bits-keep.rwc.0t0.jp/ |

### 初期構築

```bash
git clone <repo> /web/documents/BitsKeep
cd /web/documents/BitsKeep/bits-keep

composer install --no-dev
npm ci && npm run build

cp .env.example .env
php artisan key:generate
# .env の DB_* / APP_URL を環境に合わせて設定

# DB 接続先・件数・バックアップ・復元手段を確認してから実行
php artisan migrate --force

php artisan storage:link
```

### コード変更後の反映

```bash
cd bits-keep
npm run build   # 必ずビルドしてからブラウザをリロード
```

### やってはいけない操作

| 操作 | 理由 |
|---|---|
| `composer run dev` の実行 | `public/hot` が生成され画面が壊れる |
| `php artisan serve` の実行 | Apache と競合する |
| `.env` を SQLite に変えて `migrate --force` | 本番 DB が空になる（2026-04-12 事故の再現） |

### DB 変更前の必須チェック（4点）

1. 接続先確認（`.env` の `DB_HOST` / `DB_DATABASE` を目視）
2. 件数確認（`SELECT COUNT(*) FROM components` などで実データを確認）
3. バックアップ確認（ダウンロード済みのダンプがあるか）
4. 復元手段確認（`psql` で書き戻せる状態か）

4 点未確認のまま破壊的 DB 操作を実行してはならない。

### バックアップ（CLI）

```bash
# DB 取得
PGPASSWORD=postgres pg_dump -h 127.0.0.1 -p 5432 -U postgres bitskeep \
  | gzip > /tmp/bitskeep_$(date +%Y%m%d).sql.gz

# DB 書き戻し
zcat /tmp/bitskeep_YYYYMMDD.sql.gz \
  | PGPASSWORD=postgres psql -h 127.0.0.1 -p 5432 -U postgres bitskeep

# ファイル（画像・データシート）取得
tar -czf /tmp/bitskeep_files_$(date +%Y%m%d).tar.gz \
  /web/documents/BitsKeep/bits-keep/storage/app/public/

# ファイル復元
tar -xzf /tmp/bitskeep_files_YYYYMMDD.tar.gz -C /
```
