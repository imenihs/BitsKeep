# データシート解析プロンプト — BitsKeep 部品登録用

## 使い方

1. このプロンプト全文をコピーして ChatGPT（GPT-4o 推奨）に貼り付ける
2. 続けてデータシート PDF を添付して送信する
3. 返ってきた JSON を BitsKeep の「ChatGPTから貼り付け」欄に貼り付けて適用する

---

## プロンプト本文（ここからコピー）

添付した電子部品のデータシート PDF を読み取り、以下の JSON フォーマットで部品情報を出力してください。

### 出力フォーマット

```json
{
  "part_number": "型番（例: LM358N）",
  "manufacturer": "メーカー名（例: Texas Instruments）",
  "common_name": "通称・シリーズ名（例: デュアルオペアンプ）",
  "description": "部品の簡潔な説明（1〜2文、日本語可）",
  "specs": [
    { "name": "スペック名", "value": "数値または文字列", "unit": "単位（なければ空文字）" }
  ]
}
```

### 抽出ルール

**part_number**
- データシート表紙またはタイトルに記載された型番を使う
- 複数ある場合はシリーズ代表型番を1つ選ぶ

**manufacturer**
- データシートに記載されたメーカー正式名称

**common_name**
- 通称・機能名・シリーズ名（例: "オペアンプ"、"MOSFETドライバ"、"LDOレギュレータ"）
- 不明なら空文字

**description**
- 何の部品か・主な用途を1〜2文で記述
- 日本語でも英語でも可

**specs — 抽出対象（漏れなく列挙）**

以下のカテゴリから、データシートに記載されているスペックをすべて抽出すること：

1. **絶対最大定格（Absolute Maximum Ratings）**
   - 電源電圧（VCC, VDD, VS）
   - 入力電圧（VI, VIN）
   - 出力電流（IO, IOUT）
   - 消費電力（PD, Ptot）
   - 動作温度（TA, Topr）
   - 保存温度（Tstg）
   - その他記載項目すべて

2. **電気的特性（Electrical Characteristics）**
   - 入力オフセット電圧、バイアス電流
   - 電源電流（IQ, ICC, IDD）
   - 出力電圧レンジ、出力電流能力
   - 電圧利得、帯域幅（GBW）
   - スルーレート（SR）
   - CMRRおよびPSRR
   - その他記載項目すべて

3. **スイッチング特性（Switching Characteristics）**
   - 伝播遅延（tpd, tPHL, tPLH）
   - 立ち上がり・立ち下がり時間（tr, tf）
   - その他記載項目すべて

4. **推奨動作条件（Recommended Operating Conditions）**
   - 電源電圧範囲
   - 入力電圧範囲
   - 動作温度範囲
   - その他記載項目すべて

### スペック記述ルール

- `name`: 日本語を優先（例: "電源電圧"、"消費電流"）。対応する日本語が自然でなければ英語のまま可
- `value`: 典型値（typ）がある場合は典型値を使う。範囲（min〜max）の場合は `"1.8 〜 5.5"` のように記述する
- `unit`: 単位文字列（例: `"V"`, `"mA"`, `"MHz"`, `"°C"`, `"dB"`）。無次元量は `""` とする
- 条件付き値（測定条件が限定的なもの）も含める
- テーブルが複数あって同じスペック名が重複する場合、最も代表的な値を1つ選ぶ

### 出力に関する注意

- JSONのみを出力する（説明文・マークダウンコードブロック等は不要）
- 不明な項目は空文字 `""` とする（null や省略はしない）
- specs が1件もない場合は空配列 `[]` とする
- 文字コードは UTF-8、数値は文字列として格納する
