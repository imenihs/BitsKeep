# データシート解析プロンプト — BitsKeep 部品登録用

## 使い方

1. このプロンプト全文をコピーして ChatGPT（GPT-4o 推奨）に貼り付ける
2. 続けてデータシート PDF を添付して送信する
3. 返ってきた JSON を BitsKeep の「ChatGPTから貼り付け」欄に貼り付けて適用する

---

## プロンプト本文（ここからコピー）

添付した電子部品のデータシート PDF を読み取り、以下の JSON フォーマットで部品情報を出力してください。

### 出力フォーマット

```json
{
  "part_number": "型番（例: LM358N）",
  "manufacturer": "メーカー名（例: Texas Instruments）",
  "common_name": "通称・シリーズ名（例: デュアルオペアンプ）",
  "component_types": ["部品種別・機能分類の候補1", "候補2"],
  "package_names": ["詳細パッケージ候補1", "候補2"],
  "description": "部品の簡潔な説明（1〜2文、日本語可）",
  "specs": [
    {
      "name": "抽出元スペック名（データシート上の表記）",
      "name_ja": "スペック種別の日本語名",
      "name_en": "スペック種別の英語名",
      "symbol": "略記号（例: h_FE, V_CBO）。下付きは_、上付きは~で表す",
      "value_profile": "typ | range | max_only | min_only | triple",
      "value_typ": "typ値（なければ空文字）",
      "value_min": "最小値（なければ空文字）",
      "value_max": "最大値（なければ空文字）",
      "unit": "単位（なければ空文字）"
    }
  ]
}
```

### 抽出ルール

**part_number**
- データシート表紙またはタイトルに記載された型番を使う
- 複数ある場合はシリーズ代表型番を1つ選ぶ

**manufacturer**
- データシートに記載されたメーカー正式名称

**common_name**
- 通称・機能名・シリーズ名（例: "オペアンプ"、"MOSFETドライバ"、"LDOレギュレータ"）
- 不明なら空文字

**component_types**
- BitsKeep の `部品種別 / 分類候補` として使える候補名を配列で返す
- 例: `["オペアンプ", "デュアルオペアンプ"]` `["NPNトランジスタ", "汎用トランジスタ"]`
- `common_name` よりも分類名として短く、一般的な呼び方を優先する
- 候補は代表的なものから順に最大3件まで
- 候補が1件だけでも配列にする
- 不明なら空配列 `[]`

**package_names**
- データシートに記載された詳細パッケージ名を配列で返す
- 例: `["SOT-23"]` `["SOP-8", "DIP-8"]` `["TO-220", "TO-263"]`
- 表紙・注文情報・パッケージ情報・オーダリング情報に記載がある候補を代表的なものから順に列挙する
- 候補は最大5件まで
- 候補が1件だけでも配列にする
- 不明なら空配列 `[]`

**description**
- 何の部品か・主な用途を1〜2文で記述
- 日本語でも英語でも可

**specs — 抽出対象（漏れなく列挙）**

以下のカテゴリから、データシートに記載されているスペックをすべて抽出すること：

1. **絶対最大定格（Absolute Maximum Ratings）**
   - 電源電圧（VCC, VDD, VS）
   - 入力電圧（VI, VIN）
   - 出力電流（IO, IOUT）
   - 消費電力（PD, Ptot）
   - 動作温度（TA, Topr）
   - 保存温度（Tstg）
   - その他記載項目すべて

2. **電気的特性（Electrical Characteristics）**
   - 入力オフセット電圧、バイアス電流
   - 電源電流（IQ, ICC, IDD）
   - 出力電圧レンジ、出力電流能力
   - 電圧利得、帯域幅（GBW）
   - スルーレート（SR）
   - CMRRおよびPSRR
   - その他記載項目すべて

3. **スイッチング特性（Switching Characteristics）**
   - 伝播遅延（tpd, tPHL, tPLH）
   - 立ち上がり・立ち下がり時間（tr, tf）
   - その他記載項目すべて

4. **推奨動作条件（Recommended Operating Conditions）**
   - 電源電圧範囲
   - 入力電圧範囲
   - 動作温度範囲
   - その他記載項目すべて

### スペック記述ルール

- `name`: データシート上の抽出元表記。AI候補確認用であり、保存時の正本ではない
- `name_ja`: BitsKeep のスペック種別候補として使う日本語名。検索・比較したい仕様項目の粒度にする（例: `"コレクタ電流"`、`"コレクタ-ベース間電圧"`、`"直流電流増幅率"`）
- `name_en`: 同じ仕様項目の英語名（例: `"Collector Current"`、`"Collector-Base Voltage"`、`"DC Current Gain"`）
- `symbol`: 略記号（例: `"I_C"`、`"V_CBO"`、`"h_FE"`）。下付きは `_`、上付きは `~` で表す。HTMLは使わない
- `name_ja` には `最大` `最小` `typ` を入れない。値の意味は `value_profile` に入れる
- `name_ja` は `電圧` `電流` のように粗すぎず、`最大コレクタ電流` のように細かすぎない粒度にする
- データシート上の表記に `Maximum` `Minimum` `Typical` `Absolute Maximum Ratings` `Recommended` が含まれていても、スペック種別名へ混ぜない。これらは `value_profile` と抽出元 `name` だけで表す
- `DC` `pulse` `peak` `Ta=25°C` `VCE=...` などの測定条件は、同一仕様項目を別種別へ増やす理由にしない。`name_ja` / `name_en` / `symbol` は検索・比較軸として同じになる粒度を保つ
- `value_profile`: 値の意味を必ず明示する
  - `typ`: 1点の代表値
  - `range`: `min〜max` の範囲
  - `max_only`: 最大値だけ
  - `min_only`: 最小値だけ
  - `triple`: `min / typ / max` がそろっている
- `value_typ` / `value_min` / `value_max`:
  - `typ` なら `value_typ` のみ入れる
  - `range` なら `value_min` と `value_max` のみ入れる
  - `max_only` なら `value_max` のみ入れる
  - `min_only` なら `value_min` のみ入れる
  - `triple` なら3つすべて入れる
- `unit`: 単位文字列（例: `"V"`, `"mA"`, `"MHz"`, `"°C"`, `"dB"`）。無次元量は `""` とする
- `typ` と `max` が別々に記載されていて `min` が無い場合は、1件に潰さず **同じ `name` で 2 件** 出力する
- 条件付き値（測定条件が限定的なもの）も含める
- テーブルが複数あって同じスペック名が重複する場合、最も代表的な値を1つ選ぶ

### 出力に関する注意

- JSONのみを出力する（説明文・マークダウンコードブロック等は不要）
- 不明な項目は空文字 `""` とする（null や省略はしない）
- specs が1件もない場合は空配列 `[]` とする
- `component_types` と `package_names` も、候補が無ければ空配列 `[]` とする
- `specs` 内の各項目では、使わない `value_typ` `value_min` `value_max` に空文字 `""` を入れる
- 文字コードは UTF-8、数値は文字列として格納する
